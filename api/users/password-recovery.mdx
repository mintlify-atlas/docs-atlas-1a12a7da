title: 'Password Recovery'

---

## Endpoint

```
POST /api/user/pass-recovery
```

## Authentication

No authentication required.

## Request Body

<ParamField body="correo" type="string" required>
  The email address of the user requesting password recovery
</ParamField>

<ParamField body="password" type="string" required>
  The new password for the user account
</ParamField>

<Warning>
  In a production environment, this endpoint should be protected with additional security measures such as:
  - Email verification tokens
  - Rate limiting
  - Two-factor authentication
  - Password reset token expiration
</Warning>

## Response

<ResponseField name="message" type="string">
  Success message: "Contraseña cambiada" (Password changed)
</ResponseField>

<ResponseField name="userUpdate" type="object">
  Updated user object with the new password (hashed)
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST "http://localhost:8080/api/user/pass-recovery" \
  -H "Content-Type: application/json" \
  -d '{
    "correo": "juan@example.com",
    "password": "newSecurePassword123"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:8080/api/user/pass-recovery', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({

    password: 'newSecurePassword123'

console.log(data);
```

```python Python
import requests

url = 'http://localhost:8080/api/user/pass-recovery'
data = {
    'correo': 'juan@example.com',
    'password': 'newSecurePassword123'

print(response.json())
```
</RequestExample>

## Security Considerations

<Note>
  **Password Hashing**: The new password is automatically hashed using bcrypt before being stored in the database, ensuring secure password storage.
</Note>

<Warning>
  **Production Implementation**: This endpoint should be enhanced with:
  
  1. **Email Verification**: Send a verification email with a unique token before allowing password reset
  2. **Token Expiration**: Use time-limited tokens (e.g., 15-30 minutes)
  3. **Rate Limiting**: Prevent brute force attacks by limiting requests per IP
  4. **Logging**: Track password reset attempts for security auditing
  5. **Old Password Validation**: Optionally require the old password
</Warning>

## Error Handling

| Status Code | Description |
|-------------|-------------|
| 200 | Password successfully updated |
| 404 | User with the provided email not found |
| 500 | Internal server error |

<Info>
  Password updates are logged in the application with the message: `[UserController] Password updated successfully`
</Info>
