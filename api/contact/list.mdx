## Endpoint

```
GET /api/contacto
```

## Authentication

This endpoint requires authentication with admin privileges.

**Required headers:**
- `Authorization: Bearer <token>`

**Required permissions:**
- User must have admin role (`isAdmin` middleware)
- Valid JWT token (`verifyToken` middleware)

## Response

Returns an array of all contact form submissions.

<ResponseField name="contactos" type="array">
  Array of contact message objects
  
  <Expandable title="Contact message properties">
    <ResponseField name="_id" type="string">
      Unique message identifier
    </ResponseField>
    
    <ResponseField name="nombres" type="string">
      Full name of the person who submitted the contact form
    </ResponseField>
    
    <ResponseField name="correo" type="string">
      Email address of the sender
    </ResponseField>
    
    <ResponseField name="telefono" type="string">
      Phone number of the sender
    </ResponseField>
    
    <ResponseField name="asunto" type="string">
      Subject of the message
    </ResponseField>
    
    <ResponseField name="mensaje" type="string">
      Content of the contact message
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      Timestamp when the message was created (ISO 8601 format)
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      Timestamp when the message was last updated (ISO 8601 format)
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

| Status Code | Description |
|-------------|-------------|
| 401 | Missing or invalid authentication token |
| 403 | User does not have admin privileges |
| 500 | Internal server error |

<Info>
  This endpoint is typically used in admin dashboards to review and respond to customer inquiries submitted through the contact form.
</Info>

<Note>
  Contact messages are stored with automatic timestamps (`createdAt` and `updatedAt`) which can help track when inquiries were submitted and manage response times.
</Note>
