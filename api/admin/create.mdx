## Endpoint

```
POST /api/admin
```

This endpoint creates a new administrator user with a specific role (gerente, cajero, or recepcionista). The password is automatically hashed before storage.

## Request Body

### User Information

<ParamField body="nombre" type="string" required>
  First name of the administrator (will be converted to uppercase)
</ParamField>

<ParamField body="apellidoPaterno" type="string" required>
  Paternal surname (will be converted to uppercase)
</ParamField>

<ParamField body="apellidoMaterno" type="string" required>
  Maternal surname (will be converted to uppercase)
</ParamField>

<ParamField body="dni" type="string" required>
  National identification number (must be unique)
</ParamField>

<ParamField body="cumpleanios" type="string">
  Date of birth
</ParamField>

<ParamField body="genero" type="string">
  Gender of the administrator
</ParamField>

<ParamField body="img" type="string">
  Profile image URL
</ParamField>

<ParamField body="celular" type="string">
  Mobile phone number
</ParamField>

<ParamField body="direccion" type="string">
  Physical address
</ParamField>

<ParamField body="correo" type="string" required>
  Email address (must be unique)
</ParamField>

<ParamField body="password" type="string" required>
  Password (will be hashed automatically)
</ParamField>

<ParamField body="estado" type="string" default="activo">
  Account status. Possible values: `activo`, `inactivo`
</ParamField>

### Admin-Specific Information

<ParamField body="cargo" type="string" required>
  Administrative role. Possible values:
  - `gerente` - Manager
  - `cajero` - Cashier
  - `recepcionista` - Receptionist
</ParamField>

## Status Codes

| Status Code | Description |
|-------------|-------------|
| 200 | Admin user created successfully |
| 400 | Invalid request body or validation error |
| 409 | Duplicate DNI or email |
| 500 | Internal server error |

## Notes

- The password is automatically hashed using bcrypt before storage
- Names (nombre, apellidoPaterno, apellidoMaterno) are automatically converted to uppercase
- The `dni` and `correo` fields must be unique across all users
- The `rol` field is automatically set to "admin" by the discriminator
- Default estado is "activo" if not specified
