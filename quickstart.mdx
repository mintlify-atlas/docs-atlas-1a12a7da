## Prerequisites

Before you begin, ensure you have the following installed:

- Node.js (v16 or higher)
- npm or yarn package manager
- MongoDB database (local or cloud instance)
- Git

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Fisioterapia Backend API repository to your local machine:

    ```bash
    git clone <repository-url>
    cd backfisio
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required npm packages:

    ```bash
    npm install
    ```

    <Accordion title="Key dependencies installed">
      - **express** (v5.1.0) - Web framework
      - **mongoose** (v8.19.1) - MongoDB ODM
      - **jsonwebtoken** (v9.0.2) - JWT authentication
      - **passport** (v0.7.0) - Authentication middleware
      - **bcrypt** (v6.0.0) - Password hashing
      - **cors** (v2.8.5) - Cross-origin resource sharing
      - **dotenv** (v17.2.3) - Environment variable management
      - **firebase-admin** (v13.6.1) - Firebase authentication
      - **winston** (v3.18.3) - Logging
      - **morgan** (v1.10.1) - HTTP request logger
    </Accordion>
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the root directory with the following variables:

    ```bash .env
    # Server Configuration
    PORT=8080

    # Database
    MONGO_URI=mongodb://localhost:27017/fisioterapia

    # JWT Configuration
    JSON_KEY=your_secret_jwt_key_here

    # Firebase (for Google authentication)
    FIREBASE_PROJECT_ID=your_firebase_project_id
    FIREBASE_PRIVATE_KEY=your_firebase_private_key
    FIREBASE_CLIENT_EMAIL=your_firebase_client_email

    # Facebook OAuth (optional)
    FACEBOOK_ID=your_facebook_app_id
    FACEBOOK_SECRET=your_facebook_app_secret
    ```

    <Warning>
      Never commit your `.env` file to version control. Add it to your `.gitignore` file.
    </Warning>
  </Step>

  <Step title="Start MongoDB">
    Ensure your MongoDB instance is running:

    ```bash
    # If using local MongoDB
    mongod

    # Or if using MongoDB as a service
    sudo systemctl start mongod
    ```

    <Note>
      You can also use MongoDB Atlas for a cloud-hosted solution. Just update the `MONGO_URI` in your `.env` file with your Atlas connection string.
    </Note>
  </Step>

  <Step title="Start the server">
    Run the development server:

    ```bash
    # Development mode with auto-reload
    npm run dev

    # Production mode
    npm start
    ```

    You should see output like:
    ```
    info: Connected to mongoDB success
    info: Listening server on Port http://localhost:8080
    ```
  </Step>
</Steps>

## Making Your First API Request

Once the server is running, you can verify it's working by making a simple request:

### Check Server Status

```bash
curl http://localhost:8080/
```

**Response:**
```json
{
  "status": "success",
  "date": "1/5/2026, 10:30:45 AM",
  "user": "your-hostname"
}
```

### Create Your First User

Create a new user account:

```bash
curl -X POST http://localhost:8080/api/user \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Juan",
    "apellidoPaterno": "Pérez",
    "apellidoMaterno": "García",
    "dni": "12345678",
    "correo": "juan.perez@example.com",
    "password": "securePassword123",
    "celular": "987654321",
    "genero": "masculino"
  }'
```

### Login and Get JWT Token

Authenticate with the credentials you just created:

```bash
curl -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "correo": "juan.perez@example.com",
    "password": "securePassword123"
  }'
```

## Project Structure

The backend follows a modular architecture:

```
src/
├── app.js                 # Express app configuration
├── server.js             # Server entry point
├── config/
│   ├── passport.js       # Passport.js strategies
│   └── firebaseAdmin.js  # Firebase admin setup
├── controller/           # Request handlers
├── DAO/                  # Data Access Objects
├── database/
│   └── connectMongo.js   # MongoDB connection
├── helpers/
│   └── jwt.js           # JWT utilities & middleware
├── models/              # Mongoose schemas
├── router/              # Route definitions
├── utils/               # Utility functions
└── logger.js            # Winston logger configuration
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="MongoDB connection failed">
    - Verify MongoDB is running: `mongod --version`
    - Check your `MONGO_URI` in the `.env` file
    - Ensure MongoDB is accessible on the specified port (default: 27017)
    - Check MongoDB logs for errors
  </Accordion>

  <Accordion title="Port already in use">
    - Change the `PORT` in your `.env` file to a different value
    - Kill the process using port 8080: `lsof -ti:8080 | xargs kill -9`
    - Restart the server
  </Accordion>

  <Accordion title="JWT token invalid or expired">
    - Tokens expire after 1 hour - request a new token by logging in again
    - Verify the `JSON_KEY` in your `.env` matches across restarts
    - Ensure you're sending the token in the correct format: `Bearer <token>`
  </Accordion>

  <Accordion title="CORS errors">
    The API uses CORS middleware to allow cross-origin requests. If you encounter CORS issues:
    - Verify the request origin is allowed
    - Check that proper headers are being sent
    - Review CORS configuration in `src/app.js:26`
  </Accordion>
</AccordionGroup>
